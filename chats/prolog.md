## Ассистент
Программирование на **Prolog** (PROgramming in LOGic) кардинально отличается от императивных языков (Python, C++, Java). Это **декларативный** язык: вы описываете **«ЧТО»** является истинным (факты и правила), а не **«КАК»** достичь результата. Движок Prolog сам ищет решение через **унификацию** и **бэктрекинг** (возврат).

Вот краткий гид для старта.

---

#### 1. Три кита Prolog
1.  **Факты (Facts)** — базовые истины.
2.  **Правила (Rules)** — логические импликации (`Голова :- Тело`).
3.  **Запросы (Queries)** — вопросы к базе знаний.

---

#### 2. Базовый синтаксис
*   **Атомы** (константы): начинаются со **строчной** буквы: `cat`, `moscow`, `likes`.
*   **Переменные**: начинаются с **заглавной** буквы или `_`: `X`, `Who`, `_Temp`.
*   **Структуры (термы)**: `father(john, mary)`.
*   **Конец предложения**: точка `.`.
*   **И (конъюнкция)**: запятая `,`.
*   **ИЛИ (дизъюнкция)**: точка с запятой `;` (или просто несколько правил с одной головой).
*   **Отрицание (Negation as Failure)**: `\+`.

---

#### 3. Пример: Семейное дерево

```prolog
% --- ФАКТЫ ---
male(john).
male(bob).
female(mary).
female(ann).

parent(john, mary).   % Джон — родитель Мэри
parent(john, bob).    % Джон — родитель Боба
parent(mary, ann).    % Мэри — родитель Энн

% --- ПРАВИЛА ---
% отец(X, Y) истинно, ЕСЛИ X — мужчина И X — родитель Y
father(X, Y) :- male(X), parent(X, Y).

% мать(X, Y)
mother(X, Y) :- female(X), parent(X, Y).

% дедушка(X, Y) :- X отец Z, Z родитель Y
grandfather(X, Y) :- father(X, Z), parent(Z, Y).

% брат/сестра (имеют общего родителя, и это не одно лицо)
sibling(X, Y) :- parent(P, X), parent(P, Y), X \= Y.
```

---

#### 4. Как запускать запросы (в REPL / консоли)

Загрузите файл (`consult('family.pl').`) и спрашивайте:

```prolog
?- father(john, mary).      % true. (Джон отец Мэри?)
?- father(john, ann).       % false. (Джон НЕ отец Энн)

?- father(X, mary).         % X = john. (Кто отец Мэри?)
?- grandfather(X, ann).     % X = john. (Кто дедушка Энн?)

% Найти всех детей Джона
?- parent(john, Child).     
% Child = mary ;
% Child = bob.

% Найти всех братьев/сестер Энн
?- sibling(ann, Who).       % false (у Энн нет братьев/сестер в этой БЗ)
```

> **Совет:** В интерактивной консоли (SWI-Prolog, GNU Prolog) после ответа нажимайте `;` (точка с запятой), чтобы найти **следующее** решение. Точка `.` — остановить поиск.

---

#### 5. Рекурсия — главная сила Prolog
Нет циклов `for/while`. Есть только рекурсия.

**Пример: Предок (Ancestor)**
```prolog
% Базовый случай: Родитель — это предок
ancestor(X, Y) :- parent(X, Y).

% Рекурсивный случай: X предок Y, если X родитель Z, а Z предок Y
ancestor(X, Y) :- parent(X, Z), ancestor(Z, Y).
```
Запрос `?- ancestor(john, ann).` вернет `true` (Джон -> Мэри -> Энн).

---

#### 6. Работа со списками
Списки — фундаментальная структура данных. Синтаксис: `[Head | Tail]`.

```prolog
% Длина списка
len([], 0).
len([_|Tail], N) :- len(Tail, N1), N is N1 + 1.  % 'is' — арифметическое вычисление

% Членство элемента в списке
member(X, [X|_]).        % Голова совпала
member(X, [_|Tail]) :- member(X, Tail). % Ищем в хвосте

% Конкатенация (append)
append([], L, L).
append([H|T], L, [H|R]) :- append(T, L, R).
```

**Примеры запросов:**
```prolog
?- len([a,b,c], L).       % L = 3.
?- member(b, [a,b,c]).    % true.
?- append([1,2], [3,4], X). % X = [1,2,3,4].
?- append(X, Y, [1,2,3]).   % Находит ВСЕ способы разбить список:
% X = [], Y = [1,2,3] ;
% X = [1], Y = [2,3] ;
% X = [1,2], Y = [3] ;
% X = [1,2,3], Y = [].
```

---

#### 7. Арифметика: `is` vs `=`
*   `X = 2 + 2` → **Унификация**. `X` становится термом `+(2,2)`. Результат: `X = 2+2`.
*   `X is 2 + 2` → **Вычисление**. Prolog считает выражение. Результат: `X = 4`.

```prolog
?- X = 2+2.      % X = 2+2
?- X is 2+2.     % X = 4
?- 4 is 2+2.     % true
?- 2+2 = 4.      % false (структуры разные)
```

---

#### 8. Отсечение (Cut) `!` — **Используйте осторожно!**
`!` запрещает бэктрекинг (поиск альтернатив) для выборов, сделанных **левее** от него в текущем правиле. Используется для оптимизации или задания детерминизма.

```prolog
% max(X, Y, Max) — находит максимум
max(X, Y, X) :- X >= Y, !.  % Если X >= Y, ответ X, НЕ ИЩЬ ДРУГИЕ ВАРИАНТЫ
max(_, Y, Y).               % Иначе ответ Y
```
Без `!` при запросе `max(5, 3, M)` Prolog вернет `M=5`, а потом при нажатии `;` еще и `M=3` (логически неверно).

---

#### 9. Отладка и трассировка (SWI-Prolog)
Самый важный навык — читать трассировку.
```prolog
?- trace.           % Включить трассировку
?- ancestor(john, ann).
% Вывод покажет: Call, Exit, Redo, Fail для каждого предиката.
?- notrace.         % Выключить
```
Ключевые порты: **Call** (вызов), **Exit** (успех), **Fail** (провал), **Redo** (повторный поиск/бэктрекинг).

---

#### 10. Типичные ошибки новичков
1.  **Забыли точку `.`** в конце правила.
2.  **Перепутали `=` и `is`**.
3.  **Бесконечная рекурсия** (нет базового случая или он недостижим).
4.  **Опечатки в именах переменных** (Prolog создаст новую переменную, не выдаст ошибку). Используйте `singleton warnings` (в SWI: `?- set_prolog_flag(warning_singleton, true).`).
5.  **Непонимание бэктрекинга**: код выполняется не строго сверху вниз, а как поиск в дереве.

#### 11. Что попробовать следующим (Практика)
1.  **Списки**: Напишите `reverse/2`, `delete/3`, `sort/2` (merge sort или quicksort на Prolog — классика).
2.  **Графы**: Поиск пути (`path/3`), поиск в глубину/ширину.
3.  **DCG (Definite Clause Grammars)** — мощный синтаксический сахар для парсеров (`phrase/2`).
    ```prolog
    sentence --> noun_phrase, verb_phrase.
    noun_phrase --> det, noun.
    det --> [the].
    noun --> [cat].
    verb_phrase --> [sleeps].
    ```
    Запуск: `?- phrase(sentence, [the, cat, sleeps]).`
4.  **Constraint Logic Programming (CLP(FD))** — для задач на оптимизацию/расписание (`use_module(library(clpfd)).`).
